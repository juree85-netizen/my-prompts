# problem-canvas

사용자가 `/problem-canvas [문제 상황]`을 입력하거나, 사업·비즈니스 문제의 구조적 분석과 해결 방향을 요청하면 아래 절차로 실행한다.

## 실행 순서

**Phase 1 — BMC 문제 진단**: 9개 블록 중 어디에 문제가 있는지 특정  
**Phase 2 — 맥킨지 7단계**: 문제 정의 → 구조화 → 우선순위 → 분석 → 종합 → 권고안

---

## BMC 9개 블록 분류 기준

| 블록 | 코드 | 핵심 질문 |
|------|------|-----------|
| 고객 세그먼트 | CS | 누구를 위한 사업인가? |
| 가치 제안 | VP | 어떤 문제를 해결해주는가? |
| 채널 | CH | 어떻게 고객에게 전달하는가? |
| 고객 관계 | CR | 고객과 어떤 관계를 맺는가? |
| 수익 흐름 | R$ | 무엇으로 돈을 버는가? |
| 핵심 자원 | KR | 무엇이 있어야 작동하는가? |
| 핵심 활동 | KA | 무엇을 해야 하는가? |
| 핵심 파트너십 | KP | 누구와 함께해야 하는가? |
| 비용 구조 | C$ | 무엇에 돈이 드는가? |

## 출력 구조

### 0. 사업 프로파일
> [사업 유형]: B2B / B2C / 플랫폼 / 내부 서비스 / 기타  
> [문제 성격]: 수익성 / 고객 이탈 / 운영 비효율 / 성장 정체 / 파트너 문제 / 기타

### Phase 1 — BMC 문제 위치 진단

**주요 문제 블록** (직접 영향)
- [블록명]: [이 블록에서 어떤 문제가 발생하고 있는지 2~3줄]

**연쇄 영향 블록** (간접 영향)
- [블록명] ← [주요 문제 블록]과 연결되어 영향받는 이유

**BMC 진단 요약**
```
[CS] [VP] [CH]
[CR] [R$]
[KR] [KA] [KP]
[C$]
← 🔴 직접 문제 블록 / 🟡 연쇄 영향 블록 표시
```

### Phase 2 — 맥킨지 7단계 문제해결

#### Step 1. 문제 정의 (Problem Definition)
- **문제 진술**: [SMART 기준으로 — 구체적·측정가능·관련성있게]
- **문제가 아닌 것**: [범위를 좁히기 위해 명시적으로 제외]
- **성공 기준**: [이 문제가 해결됐다고 볼 수 있는 상태]

#### Step 2. 문제 구조화 (Issue Tree)
```
[핵심 문제]
├── [원인 가설 A]
│   ├── [하위 이슈 A-1]
│   └── [하위 이슈 A-2]
├── [원인 가설 B]
│   ├── [하위 이슈 B-1]
│   └── [하위 이슈 B-2]
└── [원인 가설 C]
    └── [하위 이슈 C-1]
```
> MECE 원칙: 원인 가설은 겹치지 않고 빠짐없이 구성

#### Step 3. 우선순위 설정 (Prioritization)
| 이슈 | 영향도 (H/M/L) | 해결 가능성 (H/M/L) | 우선순위 |
|------|---------------|---------------------|---------|
| [이슈 A] | H | H | ★★★ |
| [이슈 B] | H | M | ★★ |
| [이슈 C] | M | L | ★ |

#### Step 4. 분석 계획 (Work Plan)
| 이슈 | 필요한 데이터/분석 | 확인 방법 |
|------|-------------------|----------|
| [이슈 A] | [무엇을 봐야 하는가] | [어디서/어떻게] |

#### Step 5. 핵심 발견사항 (Key Findings)
- **Finding 1**: [데이터·논리 기반 발견 — "~하기 때문에 ~이다"]
- **Finding 2**: ...
- **Finding 3**: ...

#### Step 6. 종합 (Synthesis)
> **So What?** — 발견사항들이 합쳐져서 말하는 하나의 핵심 메시지  
> [3~4줄 서사: 현재 상황 → 근본 원인 → 해결 방향]

#### Step 7. 권고안 (Recommendations)
| 우선순위 | 액션 | 기대 효과 | 실행 주체 | 시점 |
|---------|------|-----------|----------|------|
| 1 | [즉시 할 것] | [~% 개선 등] | [누가] | 즉시 |
| 2 | [단기 과제] | | | 1~3개월 |
| 3 | [중기 과제] | | | 3~6개월 |

**핵심 메시지 (Elevator Pitch)**
> [의사결정자에게 30초 안에 전달할 한 문장 결론]
