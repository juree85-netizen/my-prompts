# concept-map

사용자가 `/concept-map [대상]`을 입력하거나, 특정 대상의 구조·위치·관계·맥락에 대한 설명을 요청하면 아래 절차로 실행한다.

## 실행 순서

### STEP 1 — 사전 지식 점검
대상에 대해 충분히 알고 있는지 먼저 판단한다.
- **잘 아는 경우:** STEP 2로 바로 이동
- **모르거나 불확실한 경우:** 반드시 STEP 1-A를 먼저 실행한다. 아는 척하고 넘어가지 않는다.

### STEP 1-A — 웹 검색 (모를 때만)
WebSearch 도구로 대상을 검색한다. 검색 목적은 개념도 작성에 필요한 팩트 확보다.

검색 쿼리 구성 원칙:
- 1차: `[대상]` (기본 정보)
- 필요시 2차: `[대상] 종류`, `[대상] 특징`, `[대상] 역사` 등 추가 검색

검색 후 확보한 정보를 바탕으로 STEP 2로 이동. 검색으로도 정보가 부족하면 사용자에게 배경 정보를 요청한다.

### STEP 2 — 개념도 작성
아래 출력 구조에 따라 실행한다.

## 실행 원칙

- 대상이 무엇이든 실행한다. 기업·기술·인물·음식·악기·학문·예술·개념 등 종류를 가리지 않는다.
- **섹션명은 고정하지 않는다.** 대상의 성격에 맞게 자연스러운 이름으로 바꾼다.
- 단순 정의가 아니라, 대상이 속한 **지형도** 안에서 위치와 맥락을 설명한다.
- 3·4·5번 섹션은 **MECE 원칙** — 겹치지 않고(ME) 빠짐없이(CE) 구성한다.

## 섹션 어댑테이션 원칙

각 섹션은 아래 **역할**을 유지하되, 대상에 맞게 명칭과 내용을 자유롭게 조정한다.

| 번호 | 역할 | 예시 변형 |
|------|------|----------|
| 1 | 이 대상이 속한 더 큰 범주·맥락 | 상위 개념 / 시대적 배경 / 음악 계통 / 지적 계보 |
| 2 | 왜 존재하는가 / 어떻게 등장했는가 | 존재 목적 / 등장 배경 / 발명 경위 / 주요 업적 |
| 3 | 같은 층위의 비교 대상 (MECE) | 동일 층위 비교 / 동시대 인물 / 유사 음식 / 유사 사조 |
| 4 | 대비·경쟁·영향 관계 (MECE) | 대체재·경쟁재 / 영향 관계 / 반대 개념 / 대비 장르 |
| 5 | 내부 구성·하위 요소 (MECE) | 하위 구성요소 / 주요 저작 / 하위 분야 / 재료·조리법 |
| 6 | 실제 사례·적용 맥락 | 실제 사용 사례 / 대표 작품 / 실제 적용 사례 / 대표 브랜드 |

## 출력 구조

### 0. 개체 분류
> [대상] → **[AI가 자유롭게 판단한 성격·유형]**  
> 한 줄 요약: [이게 무엇인지 한 문장]

분류는 목록에서 고르지 않는다. 대상의 본질을 가장 잘 표현하는 말로 자유롭게 기술한다.  
복합적인 경우 복수 표기도 가능하다. (예: `인물 · 경영 사상가`, `음식 · 한국 디저트`, `개념 · 물리학`)

---

### 1. [상위 맥락 — 섹션명 조정]
이 대상이 속한 더 큰 범주. 어떤 세계 안에 존재하는지 먼저 잡아준다.

### 2. [존재 목적 / 등장 배경 — 섹션명 조정]
왜 이 대상이 생겨났는가? 어떤 필요·문제·시대적 흐름에서 등장했는가?

### 3. [동일 층위 비교 — 섹션명 조정, MECE]
같은 레벨의 유사 대상들. 겹치지 않고 빠짐없이, 각 2~3줄로 비교.

### 4. [대비·경쟁·영향 관계 — 섹션명 조정, MECE]
직접 대비되거나 상호 영향을 주고받는 대상. 대상 유형에 맞게 서술.

### 5. [하위 구성요소 / 주요 요소 — 섹션명 조정, MECE]
내부를 겹치지 않고 빠짐없이 분해. 모듈·분야·재료·저작·사상 등 유형에 맞게.

### 6. [실제 사례 — 섹션명 조정]
실제 세계에서 이 대상이 어떻게 나타나는지 구체적 사례 2~3개.

### 7. 트리 구조
```
[최상위 범주]
├── [상위 범주]
│   ├── [설명 대상]  ← 여기
│   │   ├── [하위 요소 1]
│   │   └── [하위 요소 2]
│   └── [동일 층위 비교 대상]
└── [다른 상위 범주]
```

### 8. 비유
이 대상을 처음 접하는 사람도 이해할 수 있는 일상적 비유.
