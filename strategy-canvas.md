# strategy-canvas

사용자가 `/strategy-canvas [대상 또는 문제]`를 입력하거나, 문제 정의 이후 전략 방향 수립을 요청하면 아래 절차로 실행한다.

## 실행 원칙

- **문제를 이미 알고 있는 상태에서 시작한다.** 문제 정의가 안 되어 있으면 먼저 `/business-diagnosis`를 실행하도록 안내한다.
- 전략 선택지는 **MECE**로 구성한다. 겹치거나 빠지는 옵션이 없어야 한다.
- 각 선택지를 단순 나열하지 않고, **무엇을 버리는가**를 명시한다. 선택은 포기를 전제한다.
- 권고 전략은 반드시 **선택 이유 + 버린 것의 이유**를 함께 명시한다.
- 모르는 대상이나 맥락이 부족하면 WebSearch로 팩트를 먼저 확보한다.

## 실행 순서

### STEP 1 — 문제 정의 확인
사용자가 풀려는 전략적 질문이 무엇인지 확인한다.
- 명확한 경우: 바로 STEP 2
- 모호한 경우: "어떤 문제를 해결하기 위한 전략인지" 한 줄로 먼저 정리해서 확인

### STEP 2 — strategy-canvas 출력
아래 출력 구조에 따라 실행한다.

---

## 출력 구조

### 0. 전략 질문 + 맥락

> **전략 질문:** [한 줄 — 이 전략이 답해야 할 핵심 질문]
> **대상:** [기업 / 제품 / 사업부 / 프로젝트]
> **제약 조건:** [시간 / 리소스 / 시장 환경 등 전략 선택을 제한하는 요소]

**맥락 설정 — 전략 실행 전 반드시 확인한다.**

| 질문 | 내용 |
|------|------|
| 누가 결정하는가 | [이 전략의 최종 의사결정권자 — 직급·역할·관심사] |
| 어떤 제약이 있는가 | [예산 / 시간 / 조직 / 정치적 제약 — 건드릴 수 없는 것] |
| 성공 기준은 무엇인가 | [이 작업이 끝났다고 볼 수 있는 상태 — 측정 가능한 형태로] |

> 이 세 질문에 답하지 못한 채 선택지를 설계하면, 실행 불가능한 전략이 나온다.

---

### 1. 현재 상태 (As-Is)

| 항목 | 내용 |
|------|------|
| 핵심 현황 | [지금 어디에 있는가 — 수치·사실 기반] |
| 가장 큰 제약 | [이 전략에서 피할 수 없는 제한 요소] |
| 의사결정 기준 | [무엇을 최우선으로 할 것인가 — 속도 / 수익성 / 시장점유율 / 리스크 최소화 등] |
| 지금 하지 않으면 | [아무것도 안 할 때 발생하는 결과] |

---

### 2. 전략 선택지 (2~4개, MECE)

각 선택지를 독립적으로 평가한다.

#### 선택지 A — [이름]
- **핵심 방향:** [한 줄 요약]
- **핵심 가정:** [이 전략이 맞으려면 전제되어야 하는 것]
- **필요 리소스:** [인력 / 예산 / 시간 / 기술]
- **예상 임팩트:** [잘 됐을 때 어떤 결과]
- **버리는 것:** [이 전략을 선택하면 포기해야 하는 것]

#### 선택지 B — [이름]
(동일 구조 반복)

#### 선택지 C — [이름] *(필요시)*
(동일 구조 반복)

|--------|------------|-----------------|--------------------------|---------|
| A — [이름] | | | | |
| B — [이름] | | | | |
| C — [이름] | | | | |

> **핵심 Trade-off:** [이 선택지들 사이의 본질적인 긴장 관계를 한 줄로]

### 4. 권고 전략

> **선택:** [A / B / C]

**선택 이유:**
- [왜 이것인가 — 숫자·논리 기반]
- [이 선택이 의사결정 기준에 어떻게 부합하는가]

**버린 것과 이유:**
- [선택지 X를 버린 이유]
- [선택지 Y를 버린 이유]

**핵심 전제 조건:**
> 이 전략이 성공하려면 [조건 1], [조건 2]가 충족되어야 한다.
> 이 조건이 깨지면 [대안 전략]으로 전환해야 한다.

### 6. 4P 전략

권고 전략을 실행하기 위한 마케팅 믹스를 정의한다.

| 4P | 현재 (As-Is) | 전략 방향 (To-Be) | 핵심 결정 |
|----|------------|-----------------|---------|
| **Product** (제품·서비스) | [현재 무엇을 제공하는가] | [무엇을 바꾸거나 추가하는가] | [핵심 변화 한 줄] |
| **Price** (가격·수익 모델) | [현재 가격 구조] | [어떤 모델로 전환하는가] | [핵심 변화 한 줄] |
| **Place** (채널·유통) | [현재 접점] | [어느 채널을 강화·신규 개척하는가] | [핵심 변화 한 줄] |
| **Promotion** (마케팅·홍보) | [현재 커뮤니케이션] | [어떤 메시지·채널로 전달하는가] | [핵심 변화 한 줄] |

> **4P 핵심 레버:** [권고 전략에서 가장 크게 움직여야 하는 P는 무엇인가 — 그 이유]

---

### 7. GTM (Go-to-Market) 전략

시장 진입 또는 재진입 경로를 설계한다.

| 항목 | 내용 |
|------|------|
| **타겟 세그먼트** | [누구에게 먼저 — 기업 규모 / 산업 / 직무 / 지역] |
| **핵심 가치 제안** | [왜 이 고객에게 선택받는가 — 경쟁 대비 차별점] |
| **진입 채널** | [어떤 경로로 도달하는가 — 직접영업 / 파트너 / 인바운드 등] |
| **초기 트랙션 목표** | [첫 3개월 내 만들어야 할 지표 — 계약 수 / MQL / ARR 등] |
| **파트너·에코시스템** | [함께 움직일 플레이어 — SI / ISV / 채널파트너 등] |
| **킬러 레퍼런스** | [첫 번째로 잡아야 할 고객 유형 — 레퍼런스가 되는 이유] |

**GTM 단계별 모션:**

| 단계 | 타겟 | 메시지 | 채널 | 목표 지표 |
|------|------|--------|------|---------|
| **1단계 — 랜드** | [첫 진입 세그먼트] | [핵심 메시지] | [채널] | [지표] |
| **2단계 — 익스팬드** | [확장 세그먼트] | [메시지 변화] | [채널] | [지표] |
| **3단계 — 스케일** | [전체 시장] | [브랜드 메시지] | [채널] | [지표] |

> **GTM 핵심 베팅:** [이 GTM이 성공하려면 무엇이 맞아야 하는가 — 검증해야 할 핵심 가정]
