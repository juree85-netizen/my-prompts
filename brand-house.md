# brand-house

사용자가 `/brand-house [대상]`을 입력하거나, 기업·제품·아이디어를 포지셔닝·메시지화하는 요청을 하면 아래 절차로 실행한다.

## 실행 원칙

- **결론부터 출력한다.** Section 0의 Essence(한 줄)가 핵심이고 나머지는 그것을 뒷받침하는 구조.
- Brand House는 집 구조다. 지붕(Essence) → 기둥(Promise) → 기반(RTB + Values). 지붕이 흔들리면 집 전체가 흔들린다.
- Essence는 **차별점 + 가치**가 모두 담긴 한 줄이어야 한다. 설명이 필요 없어야 한다.
- "우리는 X를 하는 회사입니다"가 아니라 "고객에게 Y를 가능하게 하는 회사"로 쓴다.
- 모르는 대상은 WebSearch로 팩트 확보 후 실행한다.

### 맥락별 용어 전환

이 스킬의 구조(지붕→기둥→기반)는 B2C 브랜딩뿐 아니라 **B2B 제안·내부 보고·임원 보고**에도 동일하게 적용된다. 맥락에 따라 아래 용어로 전환해서 출력한다.

| 구조 | B2C 브랜드 맥락 | B2B·내부 보고 맥락 |
|------|--------------|-----------------|
| Section 0 | Brand Essence | 핵심 메시지 한 줄 |
| Section 1 | Target (고객) | 독자 / 의사결정권자 |
| Section 2 | Brand Promise | 우리가 전달하는 것 |
| Section 3 | RTBs | 근거·증거 |
| Section 4 | Brand Values & Tone | 전달 방식·어조 |
| Section 5 | Elevator Pitch | 30초 요약 |

> 사용자가 내부 보고·제안서·임원 발표 맥락을 언급하면 B2B 용어로 자동 전환한다.

## 실행 순서

### STEP 1 — 사전 지식 점검
- **잘 아는 경우:** 바로 STEP 2
- **모르거나 불확실한 경우:** WebSearch로 대상의 핵심 가치·타겟·포지셔닝 팩트 확보

### STEP 2 — Brand House 출력
아래 출력 구조에 따라 실행한다.

---

## 출력 구조

### 0. Brand Essence (지붕 — 핵심 한 줄)

```
┌─────────────────────────────────────────────────────┐
│                  Brand Essence                      │
│   [차별점과 가치가 모두 담긴 단 한 줄]               │
└─────────────────────────────────────────────────────┘
```

> **Essence:** "[한 줄]"

작성 기준:
- 20~35자 내외
- 읽는 순간 "아, 이게 뭔지" 알 수 있어야 함
- 경쟁자와 바꿔 써도 말이 되면 실패한 Essence

---

### 1. Target — 누구에게

| 항목 | 내용 |
|------|------|
| 핵심 대상 | [페르소나 — 직함·상황·규모] |
| 그들의 Pain | [지금 겪고 있는 불편·문제] |
| 그들이 원하는 것 | [Pain 해결 이후 얻고 싶은 상태] |
| 지금 대안 | [현재 이 문제를 어떻게 해결하고 있는가] |

---

### 2. Brand Promise — 무엇을 약속하는가

```
┌──────────────────────┬──────────────────────┐
│   기능적 약속         │   감성적 약속         │
│   (Functional)       │   (Emotional)        │
│                      │                      │
│  "무엇을 해준다"      │  "어떤 기분을 준다"   │
└──────────────────────┴──────────────────────┘
```

- **기능적 약속:** [구체적으로 무엇을 가능하게 하는가]
- **감성적 약속:** [사용 후 고객이 어떤 감정을 갖는가]
- **경쟁 대비 차별점:** [왜 다른 선택지가 아닌 이것인가]

---

### 3. RTBs — Reasons to Believe (왜 믿어야 하는가)

> 약속을 뒷받침하는 증거. 주장이 아닌 팩트·사례·수치.

| # | 근거 | 유형 |
|---|------|------|
| 1 | [팩트·수치·사례] | 데이터 / 기술 / 고객 / 수상·인증 |
| 2 | | |
| 3 | | |

---

### 4. Brand Values & Tone — 어떻게 말하는가

| 항목 | 내용 |
|------|------|
| 핵심 가치 (3개) | [이 브랜드가 가장 중요하게 여기는 것] |
| 말투·스타일 | [전문적 / 친근한 / 직관적 / 도전적 등] |
| 쓰는 표현 | [자주 쓰는 단어·어조의 예시] |
| 쓰지 않는 표현 | [이 브랜드답지 않은 말투·단어] |

---

### 5. Positioning Statement + Elevator Pitch

**포지셔닝 한 문장 (FOR-WHO-THAT-UNLIKE-OUR 구조):**
> [대상]을 위한 [카테고리]로, [핵심 약속]을 제공한다. 기존 [대안]과 달리, [핵심 차별점].

**30초 Elevator Pitch:**
> [Essence → Target의 Pain → 우리의 해결 → 증거 한 줄 → 행동 유도]
